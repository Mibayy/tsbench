How It Works

1

Index

Token Savior parses your codebase into a structural graph: functions, classes, imports, dependencies. One call to switch_project loads everything.

2

Query

Instead of reading entire files, the agent queries specific symbols: get_function_source, get_dependents, get_call_chain. Only relevant code enters the context.

3

Result

The LLM sees 81% fewer characters but gets better-structured information. Accuracy goes up because the signal-to-noise ratio improves dramatically.

Honest Limits

! MCP overhead on micro-tasks

Each TS call adds ~4K tokens of MCP schema to context. For tasks needing only a single Grep, the fixed cost makes Run B more expensive in active tokens. Affects 5 simple localisation tasks.

! 2 tasks lost on score

TASK-014 (impact analysis) and TASK-026 (dead code) score 0/2 with TS vs 2/2 baseline. Different tool outputs surfaced different symbols than expected.

! Wall time +40% on simple lookups

Single-symbol localisation is slower due to MCP round-trip latency: switch_project + find_symbol takes ~6s vs a single Grep at ~3s.

! active_tokens only -3.6%

The 81% chars reduction doesn't fully translate because cache_creation (MCP schema) dominates on short tasks. Real savings show on heavy-read and audit tasks.
