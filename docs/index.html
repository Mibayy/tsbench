Honest Limits

  • MCP overhead on micro-tasks. Each TS call adds ~4K tokens of schema to context. For tasks needing a single Grep, the fixed cost makes Run B more expensive. Affects 5 simple localisation tasks.
  • 2 tasks lost on score. TASK-014 (impact analysis) and TASK-026 (dead code audit) score 0/2 with TS vs 2/2 baseline. Different tool outputs surfaced different symbols.
  • Wall time +40% on simple lookups. Single-symbol localisation is slower due to MCP round-trip latency (switch_project + find_symbol ~6s vs one Grep ~3s).
  • active_tokens only −3.6%. The 81% chars reduction doesn't fully translate to token savings because cache_creation (MCP schema) dominates on short tasks. The real savings show on heavy-read and audit tasks.
